# Durable WebAssembly (WASM) Execution Engine

Надежный, легко переиспользуемый и производительный движок устойчивого выполнения (Durable Execution) на языке Go с использованием WebAssembly (WASM) и среды выполнения Wasmtime. Он обеспечивает отказоустойчивое исполнение бизнес-логики в изолированной песочнице с автоматическим сохранением снимков памяти (snapshotting), прозрачным восстановлением после сбоев и эффективным по памяти потоковым обменом данными.

## Особенности
- **Durable Execution (Устойчивое выполнение)**: Приостанавливает выполнение в чекпоинтах, создает снимок линейной памяти WASM и восстанавливает состояние после сбоев или перезапусков.
- **Потоковый HTTP-обмен с $O(1)$ RAM**: Передает любые данные (CSV, файлы, бинарные данные) блоками по 4 КБ напрямую в линейную память WASM и обратно через `io.Pipe`, гарантируя фиксированное потребление памяти независимо от объема передаваемых файлов.
- **Изолированная песочница WASM**: Выполняет пользовательский код в безопасной виртуальной машине на базе `wasmtime-go`.
- **Простой переиспользуемый API**: Упрощает подключение модуля благодаря экспорту ключевых хост-интерфейсов (`Engine`, `Session`, `SnapshotStore`) на корневом уровне пакета.
- **Отказоустойчивость и утечки ресурсов**: Исключает утечки благодаря переиспользованию HTTP-клиента с поддержкой контекстов (`context.Context`) и явному высвобождению ресурсов Wasmtime.

## Структура проекта
- `engine.go`: Ядро движка выполнения, управляющее жизненным циклом запусков, восстановлением памяти, вызовами хост-функций API и сетевым стримингом.
- `sqlite_store.go`: Реализация интерфейса `SnapshotStore` на базе SQLite (оптимизирована с сериализацией соединений и WAL-режимом).
- `postgres_store.go`: Реализация интерфейса `SnapshotStore` на базе PostgreSQL.
- `examples/`: Реальные примеры оркестрации бизнес-процессов:
  - `camunda/`: Оркестрация внешних задач (External Tasks) в Camunda 7 с симуляцией восстановления после сбоев.
  - `temporal/`: Долгоиграющая активность (Activity) в симулированном окружении Temporal.io с чекпоинтами.
  - `process-csv/`: Высокопроизводительный конвейер обработки и валидации CSV-потоков с потреблением $O(1)$ RAM.
  - `gotenberg-telegram/`: Загрузка документов из Telegram, конвертация в PDF через Gotenberg API и отправка обратно пользователю.
  - `durable-s3/`: Базовый пример создания снимков памяти и восстановления состояния на локальном диске.

## Начало работы

### Системные требования
- **Go**: v1.26+
- **TinyGo**: Для компиляции воркеров в WASM.
- **Docker**: Для запуска инфраструктуры тестов (Camunda, Temporal).

### Запуск тестов
Для запуска автоматических тестов ядра движка выполните:
```bash
go test -v ./...
```

## Запуск примеров

### 1. Простая демонстрация стриминга и сбоев
Запускает хост-оркестратор с использованием мок-серверов. Выполняет воркер, симулирует сбой, сохраняет снимок памяти на диск, перезапускается, восстанавливает память и доводит выполнение до конца.
```bash
make -C durable-wasm run
```

### 2. Интеграция с Camunda 7 External Tasks
Симулирует обработку заказа и проведение платежа воркером Camunda.
1. Убедитесь, что ваш контейнер Camunda 7 запущен на порту `8080`.
2. Запустите пример:
```bash
make -C durable-wasm run-camunda-example
```

### 3. Демонстрация чекпоинтов в Temporal Activity
Запускает воркер Temporal, демонстрируя пошаговое выполнение, отправку Heartbeats и продолжение выполнения с последнего сохраненного чекпоинта после сбоев.
```bash
make -C durable-wasm run-temporal-example
```

### 4. Конвейер обработки CSV в JSON ($O(1)$ RAM)
Потоково скачивает CSV-файл, валидирует и парсит строки в WASM, преобразует их в JSON и потоково отправляет результат обратно на API.
```bash
make -C durable-wasm run-csv-example
```

### 5. Конвертация документов (Gotenberg и Telegram Bot API)
Потоково считывает DOCX-шаблон из Telegram, заполняет его данными, отправляет на конвертацию в PDF в Gotenberg и загружает готовый файл обратно в чат Telegram.
```bash
make -C durable-wasm run-gotenberg-telegram-example
```

## Пример использования API

```go
package main

import (
	"context"
	"fmt"
	"path/filepath"

	"github.com/nativebpm/durable-wasm"
)

func main() {
	// 1. Инициализируем хранилище снимков (используем базу данных SQLite)
	store, err := durable.NewSqliteSnapshotStore("snapshots.db")
	if err != nil {
		panic(err)
	}
	defer store.Close()

	// 2. Загружаем и компилируем WASM-воркер, собранный через TinyGo
	engine, err := durable.NewEngine("worker.wasm", store)
	if err != nil {
		panic(err)
	}

	// 3. Выполняем логику модуля.
	// Если в БД уже есть снимок с ID "my-session-id", состояние памяти восстановится автоматически.
	ctx := context.Background()
	crashed, err := engine.Execute(ctx, "my-session-id", "run", "localhost:8080", false)
	if err != nil {
		if crashed {
			fmt.Println("Выполнение приостановлено на контрольной точке (checkpoint).")
		} else {
			fmt.Printf("Ошибка выполнения: %v\n", err)
		}
	} else {
		fmt.Println("Выполнение успешно завершено!")
	}
}
```
